Automatización con CI/CD

¿Qué es CI/CD?

La Integración Continua (CI) y la Entrega Continua (CD) representan un cambio cultural y técnico en el desarrollo de software moderno.

Concepto Enfoque Principal Resultado Típico
Integración Continua (CI) Fusión frecuente de código en un repositorio compartido. Ejecución de pruebas automáticas y validación de builds.
Entrega Continua (CD) Asegurar que el código esté siempre en un estado desplegable. Artefactos listos para producción tras pasar CI.
Despliegue Continuo (CD) Automatización total hasta el entorno de producción. Cambios visibles para el usuario final sin intervención manual.

Componentes de GitHub Actions

GitHub Actions permite automatizar flujos de trabajo directamente desde tu repositorio de GitHub.

Componente Descripción
Workflow Proceso automatizado configurable (archivo .yml).
Events Actividades que disparan el flujo (push, pull_request, schedule).
Jobs Conjunto de pasos que se ejecutan en el mismo corredor (runner).
Steps Tareas individuales como ejecutar comandos o usar una "Action".

Ejemplo de Configuración (.yml)

Este archivo debe ubicarse en .github/workflows/main.yml para activarse automáticamente.

name: Node.js CI

on:
  push:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install dependencies
        run: npm install
      - name: Run tests
        run: npm test
